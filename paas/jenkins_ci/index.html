Jenkins- Continuous Integration Server

Jenkins is our open source Continuous Integration tool used for BUILD and Release Jobs.

It is an open source product with customisabel plugins which enables it to integrate it with multiple SCMs and run various types of build jobs. Each check in is verified with an automated build allowing early detection of problems. It has the ability to send the status of build and release jobs over an email or in HipChat rooms.

ex: connectivity with git.mercurial,Subversion etc

ex: Jobs like; ant,Shell,Windows batch commands, gradle etc.

Design Principle of Jenkins

Jenkins is integrated with GitHUb and has been configured to run two ways:

  1. Build jobs for a specific branch
  2. Release jobs when there is a commit in the master branch

Build jobs for a specific branch

This allows multiple developers to run build jobs for their specific branches.

Branch Selector is a jenkins plugin installed for the same.

As part of the Build for microservices, we are running “gradle clean build command” which builds the docker image for the microservice and also runs UNIT and Components Tests for it.

This is followed by static analysis of the code and publishing unit tests results.

Release Jobs

As part of the release jobs, apart from the build, Jenkins also runs the following jobs

  1. gradle clean publishTestComponent –refresh-dependencies - this is to publish the integration tests as artifacts in the Nexus repo

  2. gradle clean build testAndBurn –refresh-dependencies- as part of this job, after building the docker image, Call Composer- builds a dev environment and deploy all the components and then run integration test against the component.

  3. gradle clean pushDockerImage caboodle:publish –refresh-dependencies- As part of this job, the Image is published to caboodle(Artifact Repository)

  4. “component_version=grep 'version' gradle.properties |awk -F= '{ print $2 }' | awk -F.0- '{ print $1 }'

   component_version=2.0
   tag_id=$component_version.$BUILD_NUMBER
   echo $tag_id
   git tag -a $tag_id -m "Jenkins Build"
   git push origin $tag_id" - This task is tagging GitHub back with the build number
