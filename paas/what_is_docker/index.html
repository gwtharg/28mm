What is Docker?

Docker is an Open Source software development platform. It’s main benefit is to package applications into “containers”, allowing them to be portable and lightweight among any system running the Linux OS. Container technology has been around for a few years, with Solaris Zones, IBM Wpars etc, but the hype around Dockers approach to containers has pushed this technology to the forefront in the last couple of years.

Benefits of Docker Containers

Scalability

Container technology allows for a much larger scale of applications in virtualised environment. With a perfectly tuned container, you can have 4-6 times the number of application instances as you can using VM’s on the equivalent hardware. This means you reducde the overhead of running a VM, leaving you a small container running your application.

Portability

Containers elimates the “it works ok on my machine” problem as the same docker container image will run across all Linux based OS platforms.

Speed of development

Developers can quickly develop, test and deploy docker containers. They don’t need to worry about setting up complex development environments.

Can be pre-packaged

Application providers are now starting to ship their applications in pre-built containers. Gone are the days of building an environemnt then having to install your application, it comes already packaged for you.

For example, let’s dowmload a pre-packaged HTTP server from redhat and launch the docker image:

How to build a docker image

The easiest way to build a docker image is to use an existing image and layer your changes on top of it. This then creates a newer version with your updates applied. Let’s take the above pre-packaged http server as an example.

We are going to place a simple Hello World index page into the image to display when you access the URL

To do this, we create a Dockerfile. this file tells the docker build process what to do. So in our case, we are going to say where the original docker image should come FROM and then COPY in our new index file:

cat Dockerfile

FROM registry.access.redhat.com/rhscl/httpd-24-rhel7

COPY index.html /opt/rh/httpd24/root/var/www/html/

cat index.html

<P>Hello world

Once we have our Dockerfile in place, we tell docker to build our image, and give it a new name:

docker build -t new_http_image .

docker run -p 8080:8080 new_http_image
